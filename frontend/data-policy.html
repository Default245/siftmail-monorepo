Client Data Storage Policy

This page explains what Sift Mail stores and how we secure your data.

Data Stored

  • OAuth tokens (refresh + access) for ongoing Gmail API access.
  • User profile basics from Google (email and display name).
  • Preference settings (rules, allow/deny lists, digest prefs).
  • Metadata logs (message IDs + classification outcomes for undo + quality).

Data Not Stored

  • We do not store full email bodies by default.
  • No attachments or inline images are retained.

Security

  • Encryption at rest & TLS in transit.
  • No third‑party sharing without explicit consent.
